# Quiz — Модуль прохождения теста

Тестовое задание для QazaqSoft.

## Запуск

Нужен локальный сервер, так как приложение грузит JSON через fetch:

```bash
npx serve .
# или
python -m http.server 8000
```

## Тесты

```bash
node tests.js
```

## Структура

```
quiz/
  index.html
  app.js
  styles.css
  tests.js
  data/
    questions.json
  README.md
```

## Что реализовано

- Загрузка вопросов из data/questions.json
- Навигация Назад / Далее / Завершить
- Таймер обратного отсчёта, при 0 — автозавершение
- Восстановление выбранного ответа при возврате к вопросу
- Экран результата: счёт, процент, статус
- Просмотр ответов с подсветкой правильных и ошибочных
- Сохранение в localStorage, восстановление после перезагрузки
- Перемешивание вопросов и вариантов
- Unit-тесты на логику QuizEngine
- Адаптив, управление с клавиатуры, aria-атрибуты

## Детали реализации

Перемешивание вариантов сохраняет связь с правильным ответом: вместо хранения
фиксированного индекса при перемешивании запоминается значение правильного варианта
и индекс пересчитывается. Перемешанный порядок вопросов и вариантов сохраняется в
localStorage вместе с ответами, иначе после перезагрузки сохранённый индекс указывал
бы на другой вариант.

## Что стоит улучшить в реальном проекте


### 1. Бэкенд с авторизацией (критично для продакшена)
В текущей реализации все данные (правильные ответы, прогресс пользователя) хранятся на клиенте. В реальном проекте нужно:
- Перенести проверку ответов на бэкенд (через API)
- Добавить авторизацию (JWT или сессии)
- Сохранять результаты тестов в БД (PostgreSQL)
- Исключить хранение правильных ответов в клиентском JSON

### 2. Аналитика и мониторинг
Сейчас нет сбора метрик. Для бизнеса важно отслеживать:
- Среднее время прохождения теста
- Какие вопросы вызывают наибольшие затруднения (по времени и количеству ошибок)
- Процент прохождения теста с первой попытки
- Retention пользователей

### 3. Админ-панель
Необходим интерфейс для создания/редактирования вопросов без правки JSON. Можно реализовать на React/Vue или использовать существующие решения (Strapi, Directus).

### 4. Типизация и тестирование
Хотя есть unit-тесты, стоит добавить:
- TypeScript для типизации (уменьшит количество багов)
- E2E-тесты (Playwright/Cypress) для проверки полного сценария прохождения теста
- Интеграционные тесты для API (если добавить бэкенд)

### 5. Доступность (a11y)
Базовая доступность уже есть (aria-label, клавиатура). Можно улучшить:
- Live-регионы для динамических обновлений (например, при смене вопроса)
- Правильные роли для элементов (radiogroup, radio)
- Контрастность цветов под WCAG 2.1

### 6. Интернационализация (i18n)
Поддержка казахского, русского, английского языков. Использовать i18next или react-i18next.

### 7. Оптимизация производительности
- Ленивая загрузка вопросов при их большом количестве (пагинация или виртуализация)
- Debounce для сохранения в localStorage (сейчас сохраняется при каждом действии)

### 8. DevOps
- Docker-контейнеризация
- CI/CD через GitHub Actions (автоматический деплой на Vercel/Render)
- Автоматические проверки (линтеры, форматтеры, тесты) при пуше
